# 赛道一参赛 + 毕设「两用」对齐建议（基于任务书草稿）

> 任务书草稿：`docs/任务书草稿.docx`
>
> 赛道一要求：`docs/第四届“凌特杯”赛项说明.pdf`

## 1. 一句话结论

你的毕设题目与赛道一高度同题：**“跳频 + 抗干扰调制/编码 + 图像可靠传输”**天然可以复用同一套系统、同一套仿真数据与图表；需要补齐的主要是：**随机脉冲干扰信道模型**、**资源约束指标（带宽/时宽/功率）**、以及**eLabRadio 平台迁移/复现路线**。

---

## 2. 任务书内容与赛道一要求的映射

### 2.1 可直接一套做完（强复用）

- 文献调研：隐蔽通信、FHSS、抗干扰调制/编码（任务书“任务1/内容一”）⇔ 赛道一“信道建模与干扰分析/抗监听技术”
- 系统方案：信源处理、跳频序列、调制解调与同步（任务书“任务2/内容二”）⇔ 赛道一“调制与同步模块设计/系统架构”
- 仿真平台：端到端链路与联调（任务书“任务3/内容三”）⇔ 赛道一“eLabRadio 平台实现系统仿真”
- 性能验证：BER/PSNR + 干扰下对比（任务书“任务4/内容四”）⇔ 赛道一“仿真数据（误码率、图像质量、频谱图等）”

### 2.2 建议补强（赛道一评分点更敏感）

任务书中提到 AWGN/单音干扰较多；赛道一的核心是“随机脉冲干扰”。建议在任务书/论文中明确采用一种或两种常用模型：

- Bernoulli-Gaussian（BG）脉冲噪声（你现在的 MATLAB 基线已用）
- Middleton Class-A（可作为增强/对比）

并补齐“干扰识别与抑制”的定量对比（如 blanking/clipping/擦除+软译码/交织深度等）。

## 3. 「一套工程，两份交付」怎么组织

### 3.1 工程输出（唯一数据源）

以一套仿真脚本/参数为唯一事实来源，批量产出：

- BER–Eb/N0 曲线（含不同干扰参数）
- PSNR/SSIM–Eb/N0 曲线（含重建图）
- 频谱 PSD、99% 占用带宽（OBW）、频谱效率
- 突发时宽（每帧符号数/时间占用）与码率/吞吐

这些图表可以同时进“赛道报告”和“毕设论文”，只是在**叙事角度**不同（见 3.2/3.3）。

### 3.2 赛道报告怎么写（偏工程交付）

围绕赛道一评分维度组织：

- 技术完整性：端到端模块齐全（信源/信道/调制/同步/抗干扰）
- 抗干扰性能：随机脉冲干扰下的 BER、图像可辨识（附对比）
- 资源约束：带宽占用、发射时间（突发）、功率/能量指标（用归一化或相对量也可）
- 可执行性：平台实现/仿真可复现（参数表 + 关键截图）

### 3.3 毕设论文怎么写（偏学术论证）

在“赛道报告”的基础上增加：

- 更系统的相关工作综述与分类（FH 序列、LPI/LPII 思路、脉冲噪声建模）
- 关键模块的数学建模/推导与复杂度分析（例如阈值策略、交织/FEC 的机理解释）
- 参数敏感性分析与消融实验（为什么选这个方案，而不是只给结果）
- 合规与伦理讨论（通信安全/隐私保护的边界、法规约束），把“隐蔽”叙述为**抗干扰与隐私保护的工程需求**，避免表述歧义

## 5. 代码层面的建议（方便两用）

建议每个“论文模块”对应一个独立 m 文件，并用统一的参数结构体驱动批量实验。

你当前代码结构示例（都在 `src/link/`）：

- 信道与抗干扰：`channel_bg_impulsive.m`、`mitigate_impulses.m`
- 同步：`frame_sync.m`
- 编译码/交织：`fec_encode.m`、`fec_decode.m`、`interleave_bits.m`
- 指标：`estimate_spectrum.m`、`image_quality.m`

后续若加入跳频，可把“跳频/解跳”作为独立模块插在“调制后、信道前”与“信道后、解调前”，并把“同步”拆成（帧/符号/载波）三个可单测模块，利于论文和报告复用同一套图表。

## 6. 参赛要求用 eLabRadio（专有平台）怎么办？

结论：**不冲突**。把 MATLAB 当作“参考模型/金标”，把 eLabRadio 当作“参赛交付与平台验证”，反而能让毕设更完整（多了工程复现与一致性验证这一章）。

### 6.1 推荐的两层实现策略

- **MATLAB（参考模型）**：用于算法推导、参数扫描、快速迭代、产出“应当达到的曲线/图像/频谱”。
- **eLabRadio（参赛实现）**：按赛题要求搭建同等链路（或等效链路），用同一组输入与参数复现实验结果，并输出赛题要求的图表与数据。

写在报告/论文里时，可以描述为：*“算法在 MATLAB 完成建模与验证；在 eLabRadio 竞赛平台完成系统级仿真复现，二者结果一致性作为工程正确性验证。”*

### 6.2 平台迁移不要“整套重写”，按接口逐段对齐

最省力的方式不是一次把所有东西搬过去，而是按“信号接口”拆段验证：

1) **定义每段接口**（推荐 3 个层次任选其一）  
- bit 接口：`bits`（最适合验证编译码/交织/扰码）  
- symbol 接口：`sym`（最适合验证调制/抗干扰/同步）  
- sample 接口：`I/Q`（最适合验证频谱占用、带宽、突发时宽）

2) **在 MATLAB 产出金标向量**（输入/输出对）  
例如：给定同一幅图像与同一随机种子，保存 `headerBits/payloadBits`、`codedBits`、`txSym`、以及关键参数表。

3) **在 eLabRadio 喂入同样的输入与参数**  
- 如果 eLabRadio 支持 File Source：直接喂 `bits/sym/IQ` 文件  
- 如果只能从块内部产生：确保 PN、交织、FEC、多项式、种子等参数与 MATLAB 一致

4) **逐段比对/逐段替换**  
- 先保证“无干扰”链路一致  
- 再打开“随机脉冲干扰 + 抗干扰”  
- 最后加“跳频/突发/加密”等隐蔽性模块

这样做的好处：你不会陷入“eLabRadio 里整体跑不起来却不知道哪段错”的黑箱。

### 6.3 对毕设的写作收益（专有平台反而加分）

专有平台最大的担心是“可复现性/可审查性”。处理方式是把“可复现”做成两层：

- **算法可复现（MATLAB）**：附录给出关键公式、参数表、伪代码、以及可运行脚本（这是导师/评阅最关心的）。
- **平台可复现（eLabRadio）**：提交平台工程文件 + 截图 + 参数导出表；论文中说明平台与版本、链路结构、以及如何运行复现实验即可。

### 6.4 你现在应该立刻加的一条任务（同时满足赛题与毕设）

把任务书里的“仿真平台搭建”补一句更工程化的表述：

- “建立 MATLAB 参考模型，并在 eLabRadio 竞赛平台复现同等链路；用一致性对比验证工程正确性。”

这句话能同时解释：你为什么用 MATLAB（不浪费），以及你如何满足赛题“必须 eLabRadio”的要求。
