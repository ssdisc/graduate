# 项目理解报告：抗脉冲干扰无线图像传输系统

## 项目背景

这是一个为**第四届"凌特杯"赛道一（初赛）**设计的MATLAB端到端基线系统，用于实现**抗脉冲干扰的无线图像传输**。该项目同时可作为**本科毕业设计**使用。

### 核心目标
1. 在强脉冲噪声环境下实现可靠的图像传输
2. 使用机器学习技术检测和抑制脉冲干扰
3. 评估系统的抗干扰性能（BER、PSNR、SSIM）
4. 分析系统的隐蔽性和安全性（窃听者Eve、监视者Warden）

---

## 系统架构概览

### 完整信号处理流程

```
发送端 (TX):
图像源 → 混沌加密 → 载荷比特 → 帧头构建 → PN扰码 → 卷积编码
→ 块交织 → BPSK调制 → 跳频调制 → 添加前导 → 发送符号

信道:
AWGN + 伯努利-高斯脉冲噪声
y = x + n_AWGN + I·n_impulse
I ~ Bernoulli(p=1%), n_impulse ~ N(0, 50×N0)

接收端 (RX):
接收符号 → 帧同步 → 数据提取 → 跳频解调 → 脉冲抑制(6种方法)
→ 软解调 → 解交织 → Viterbi软判决译码 → 解扰码 → 帧解析
→ 混沌解密 → 图像重建 → DnCNN降噪(可选) → 质量评估
```

## 核心技术模块详解

### 1. 混沌加密系统（新增功能）

**文件位置**: `src/link/chaos_encrypt.m`, `chaos_decrypt.m`, `arnold_transform.m`, `chaos_generate.m`

**加密流程**:
```
原始图像
  ↓
Arnold变换（像素位置置乱，迭代N次）
  ↓
Logistic混沌序列生成（x_{n+1} = μ·x_n·(1-x_n)）
  ↓
扩散操作（像素值与混沌序列异或）
  ↓
加密图像
```

**关键参数**:
- `arnoldIter`: Arnold变换迭代次数（默认5次）
- `chaosMethod`: 混沌映射类型（'logistic', 'henon', 'tent'）
- `chaosParams`: 混沌参数（密钥）
  - `mu`: Logistic参数 (3.57 < mu <= 4)
  - `x0`: 初始值（密钥的一部分）

**安全性**:
- 即使Eve成功解码比特流，没有混沌密钥也无法恢复原始图像
- Arnold变换提供空间域置乱
- 混沌序列提供值域扩散
- 密文反馈机制增强抗差分攻击能力
- 双重保护机制

**实现细节** (src/link/chaos_encrypt.m:69-91):
```matlab
% 密文反馈扩散
for round = 1:enc.diffusionRounds
    key = keyStream(keyStart:keyEnd);
    encrypted = zeros(nPixels, 1, 'uint8');
    prevCipher = key(1);  % 初始值

    for i = 1:nPixels
        % 异或：明文 XOR 密钥 XOR 前一密文
        encrypted(i) = bitxor(bitxor(imgVec(i), key(i)), prevCipher);
        prevCipher = encrypted(i);  % 密文反馈
    end

    imgVec = encrypted;
end
```

### 2. 脉冲抑制技术（核心创新）

**文件位置**: `src/link/mitigate_impulses.m`

系统实现了**6种脉冲抑制方法**，可并行对比性能：

| 方法 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **none** | 不处理 | 基准对比 | 性能最差 |
| **blanking** | 幅度超过阈值→置零 | 简单快速 | 损失有用信号 |
| **clipping** | 幅度超过阈值→削波 | 保留部分信息 | 引入非线性失真 |
| **ml_blanking** | 逻辑回归检测+置零 | 检测更准确 | 仍是硬判决 |
| **ml_cnn** | 1D CNN检测+软抑制 | 软判决，输出可靠性 | 需要训练 |
| **ml_gru** | GRU检测+软抑制 | 利用时序信息 | 计算复杂度高 |

#### ML模型的关键创新：软符号替换

传统方法（blanking/clipping）是**硬判决**：
```
if |r| > threshold:
    r_clean = 0  # 完全置零
```

ML方法（CNN/GRU）是**软判决**：
```
[p_impulse, reliability, clean_real, clean_imag] = CNN(r)
r_clean = (1 - p_impulse) × r + p_impulse × (clean_real + j×clean_imag)
```

**输出4个值**:
1. `p_impulse`: 脉冲概率（0-1之间）
2. `reliability`: 判决可靠性权重
3. `clean_real`: 清洁符号实部估计
4. `clean_imag`: 清洁符号虚部估计

**优势**:
- 不是简单的"是/否"判决，而是给出概率
- 提供可靠性权重，传递给后续Viterbi译码器
- 估计清洁符号值，而不是简单置零

**实现细节** (src/link/mitigate_impulses.m:58-78):
```matlab
case "ml_cnn"
    % 1D CNN带软输出
    model = mit.mlCnn;
    [mask, rel, cleanSym, pImp] = ml_cnn_impulse_detect(r, model);

    % 对检测到的脉冲使用清洁符号，否则使用原始符号
    rOut = r;
    if model.trained
        % 混合：使用脉冲概率加权的清洁估计
        rOut = (1 - pImp) .* r + pImp .* cleanSym;
    else
        % 未训练：退回到置零
        rOut(mask) = 0;
    end
    reliability = double(gather(rel));
```

#### CNN模型架构

**文件**: `src/link/ml_cnn_impulse_model.m`

```
输入: 4个特征 [real, imag, magnitude, magnitude_diff]
  ↓
Conv1D(16滤波器, kernel=5) → ReLU
  ↓
Conv1D(32滤波器, kernel=3) → ReLU
  ↓
全连接层(4输出)
  ↓
输出: [p_impulse, reliability, clean_real, clean_imag]
```

**特征提取** (`ml_cnn_features.m`):
1. `real(r)`: 实部
2. `imag(r)`: 虚部
3. `|r|`: 幅度
4. `||r_i| - |r_{i-1}||`: 幅度差分（检测突变）

### 3. DnCNN图像降噪（新增功能）

**文件位置**: `src/link/ml_image_denoise.m`, `ml_image_denoise_model.m`

**架构**（残差学习）:
```
输入图像(64×64 patch)
  ↓
Conv(64, k=3) → ReLU
  ↓
[Conv(64, k=3) → BatchNorm → ReLU] × 15层
  ↓
Conv(1, k=3) → 残差输出(噪声估计)
  ↓
去噪图像 = 输入图像 - 残差
```

**关键设计**:
- **残差学习**: 网络学习预测噪声，而不是直接预测清洁图像
- **滑动窗口**: 将大图像分成64×64小块处理
- **批归一化**: 加速训练，提高泛化能力

**使用场景**:
- 在接收端图像重建后，进一步降噪
- 可选功能（`p.denoise.enable = true`）
- 测量降噪前后的PSNR/SSIM增益

### 4. 跳频扩频系统

**文件位置**: `src/link/fh_generate_sequence.m`, `fh_modulate.m`, `fh_demodulate.m`

**原理**:
```
发送端: s_tx(n) = s(n) × exp(j·2π·f_k·n)
接收端: s_rx(n) = r(n) × exp(-j·2π·f_k·n)
```

**参数配置**:
```matlab
p.fh.enable = true;           % 启用跳频
p.fh.nFreqs = 8;              % 8个跳频频点
p.fh.symbolsPerHop = 64;      % 每64个符号跳一次
p.fh.freqSet = linspace(-0.35, 0.35, 8);  % 归一化频率
```

**优势**:
1. **抗窄带干扰**: 干扰只影响部分跳频周期
2. **低截获概率**: 信号能量分散在多个频点
3. **隐蔽性增强**: Eve不知道跳频序列则无法解调

### 5. 信道编码与交织

**卷积编码** (`fec_encode.m`):
- 码率: 1/2（每1比特编码为2比特）
- 生成多项式: [171, 133]（八进制）
- 约束长度: 7

**Viterbi译码** (`fec_decode.m`):
- 支持硬判决和软判决（3比特量化）
- 软判决利用脉冲抑制模块输出的可靠性权重
- 性能优于硬判决约2-3dB

**块交织** (`interleave_bits.m`):
- 将连续比特打散，抗突发脉冲
- 可配置交织深度（`p.interleaver.nRows = 64`）
- 深度越大，抗突发能力越强，但时延增加

### 6. 隐蔽性与安全性分析

#### 窃听者Eve仿真

**文件**: `simulate.m` (lines 76-100)

**场景设置**:
- Eve的SNR比Bob差（例如 -6dB）
- Eve对密钥的知识可配置：
  - `scrambleAssumption`: "known" / "none" / "wrong_key"
  - `fhAssumption`: "known" / "none" / "partial"
  - `chaosAssumption`: "known" / "none" / "wrong_key"

**评估指标**:
- Eve的BER曲线
- Eve重建图像的PSNR/SSIM
- 与Bob的性能差距

#### 监视者Warden检测

**文件**: `warden_energy_detector.m`

**原理**: 能量检测器
```
H0: 只有噪声（无信号）
H1: 有信号传输

检测统计量: T = Σ|r_i|²
```

**评估指标**:
- `Pfa`: 虚警概率（误判为有信号）
- `Pd`: 检测概率（正确检测到信号）
- `Pe`: 错误概率

## 技术亮点与创新

### 1. 多层安全保护
- **PN扰码**: 轻量级加密，白化比特流
- **跳频**: 频域扩散，Eve不知道序列无法解调
- **混沌加密**: 图像级加密，即使解码比特也无法恢复图像

### 2. 软判决全链路
- 脉冲抑制输出软可靠性
- 软解调输出LLR
- Viterbi软判决译码
- 性能优于硬判决2-3dB

### 3. ML模型输出4值
- 不是简单的"有/无"脉冲判决
- 输出概率、可靠性、清洁符号估计
- 支持软符号替换

### 4. 残差学习降噪
- DnCNN学习预测噪声而非图像
- 训练更稳定，泛化能力更强
- 滑动窗口处理任意大小图像

### 5. 模块化设计
- 每个功能独立文件
- 参数集中配置
- 易于扩展和修改

## 常见问题

### Q1: 为什么需要混沌加密？
**A**: 即使Eve成功解码比特流，没有混沌密钥也无法恢复原始图像。提供图像级的安全保护。

### Q2: 软判决比硬判决好在哪里？
**A**: 软判决保留了概率信息，Viterbi译码器可以利用这些信息做出更好的判决，性能提升2-3dB。

### Q3: 为什么ML方法比传统方法好？
**A**:
- 传统方法（blanking/clipping）基于简单阈值，容易误判
- ML方法学习脉冲的统计特性，检测更准确
- ML方法输出软可靠性，而不是硬判决

### Q4: DnCNN为什么用残差学习？
**A**:
- 直接预测清洁图像很难（目标复杂）
- 预测噪声更容易（噪声相对简单）
- 残差学习训练更稳定，收敛更快

### Q5: 跳频如何提高隐蔽性？
**A**:
- 信号能量分散在多个频点，单个频点能量低
- Eve不知道跳频序列，无法正确解调
- 即使Eve知道有信号，也难以截获内容

这是一个**完整、可运行、模块化**的抗脉冲干扰无线图像传输系统，具有以下特点：

✅ **完整性**: 从图像输入到图像输出的端到端链路
✅ **创新性**: 机器学习脉冲抑制 + 混沌加密 + DnCNN降噪
✅ **可扩展性**: 模块化设计，易于添加新功能
✅ **可评估性**: 多种性能指标（BER/PSNR/SSIM）
✅ **安全性**: 多层保护（扰码/跳频/混沌加密）
✅ **可对比性**: 6种脉冲抑制方法并行对比

该系统可直接用于：
1. **赛道一参赛**: 提交完整的MATLAB仿真结果
2. **毕业设计**: 作为论文的技术实现部分
3. **学习研究**: 理解通信系统和机器学习的结合
