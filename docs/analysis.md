### 10. 信道模型（任务三）

| 要求 | 实现 | 评价 |
|------|------|------|
| AWGN 信道模拟热噪声 | BG 模型中的背景高斯分量 | **符合** |
| **瑞利衰落信道模拟多径效应** | **静态多径抽头**（固定增益 [0, -12, -18] dB，固定相位） | **不符合** |
| BG 脉冲噪声 | `impulseProb=0.01`, `impulseToBgRatio=50` | **符合** |
| 单音干扰 | `singleTone` 配置 | **符合** |
| 部分频带噪声干扰 | `narrowband` 配置（带通滤波成形） | **符合** |
| 扫频干扰 | `sweep` 线性 chirp | **符合** |

**关键偏差——瑞利衰落 vs 静态多径**：

开题报告明确写"构建**瑞利衰落信道**来模拟多径效应"。实际实现 [channel_bg_impulsive.m:43-59](src/channel/channel_bg_impulsive.m#L43-L59) 使用**固定增益、固定相位的静态抽头滤波器**，不是瑞利衰落：

```matlab
amp = 10.^(gDb/20);
cplxAmp = amp .* exp(1j*phase);  % phase是固定值或一次性随机
xCh = filter(mpTaps, 1, xCh);   % 静态FIR滤波
```

**分析**：
- 瑞利衰落需要时变信道抽头（每个符号时刻各径增益独立地服从 Rayleigh 分布）
- 当前实现是**静态频选信道**（ISI），不是**时变衰落信道**
- 对于慢跳频系统，如果跳频速率远大于信道相干时间，每跳可以视为平坦衰落，但当前实现并没有这个时变机制
- **实际影响**：开题报告的多径要求未被满足，但对于"抗脉冲干扰"这个核心主题来说，BG 脉冲模型比多径衰落更重要。静态多径仍然测试了 ISI 抗干扰能力。

### 11. 接收同步（任务三）

| 要求 | 实现 | 评价 |
|------|------|------|
| 利用**混沌序列的自相关尖峰**特性实现粗同步 | 默认用 **PN 前导**（m-序列），差分相关粗同步 | **偏差但更合理** |
| 设计**延迟锁相环**维持时钟对齐 | [timing_dll_sync.m](src/sync/timing_dll_sync.m) 实现了 DLL，但**默认禁用** | **部分符合** |
| FFT 变换域滤波或自适应陷波器剔除窄带干扰 | 在 mitigation 阶段实现，非 sync 阶段 | **符合**（位置不同） |

**关键偏差——前导类型**：

开题报告专门强调"利用混沌序列的自相关尖峰特性来实现粗同步"，这是将混沌理论应用于同步的一个创新点。但实际实现：
- `p.frame.preambleType = "pn"` → 默认使用 m-序列 PN 前导
- 代码支持 `"chaos"` 前导，但不是默认选项

**分析**：
- m-序列的自相关特性（零旁瓣）**优于**混沌序列（有限精度下自相关旁瓣较高）
- 从工程角度看 PN 前导更稳定可靠，**实现选择更合理**
- 但从论文创新性角度看，开题报告将"混沌前导同步"作为亮点之一，应至少做对比实验

**DLL 默认禁用**：
- `p.rxSync.timingDll.enable = false`
- 开题报告明确要求"设计延迟锁相环维持时钟对齐"
- 代码实现了完整的 DLL，但默认关闭，实际同步依赖帧同步时的分数定时估计+PLL
- **建议**：至少在论文中做 DLL 开/关的对比实验

### 核心结论

1. **整体完成度很高**：开题报告五大任务的核心内容均已实现，且大量超出
2. **最大偏差**是**瑞利衰落**→静态多径，以及**混沌前导**默认未启用，但这两个偏差都有合理的工程理由
3. **实现比开题报告更合理的地方**主要是：分包传输、波形成型、交织器、扰码——这些是开题报告遗漏但工程必需的环节
4. **建议补充**：混沌序列统计特性分析模块（开题报告的理论创新点之一）；混沌 vs PN 前导的同步性能对比实验
